SQL Constraints

NOT NULL

The NOT NULL constraint enforces a column to NOT accept NULL values.

While creating table
CREATE TABLE Persons (
ID int NOT NULL,
LastName varchar(255) NOT NULL,
FirstName varchar(255) NOT NULL,
Age int
);
Add to existing table
ALTER TABLE table_name MODIFY column_name datatype NOT NULL;
Drop NOT NULL
ALTER TABLE table_name MODIFY column_name datatype NULL;
UNIQUE

The UNIQUE constraint ensures that all values in a column are different.

Adding to an existing table
create table studs1(
regno number(12),
name varchar2(10),
city varchar2(7),
constraint mynuique unique(name)
);
Adding to existing table
ALTER TABLE table_name ADD CONSTRAINT MyUniqueConstraint UNIQUE(column1, column2...);
Droping Unique Contraint
ALTER TABLE table_name DROP CONSTRAINT MyUniqueConstraint;
DEFAULT

The DEFAULT constraint is used to provide a default value for a column.

While creating a table
CREATE TABLE CUSTOMERS(
ID INT NOT NULL,
NAME VARCHAR (20) NOT NULL,
AGE INT NOT NULL,
ADDRESS CHAR (25) ,
SALARY DECIMAL (18, 2) DEFAULT 5000.00,
PRIMARY KEY (ID)
);
Add to and existing table
ALTER TABLE CUSTOMERS MODIFY SALARY DECIMAL (18, 2) DEFAULT 5000.00;
Drop the default contraint
ALTER TABLE constomers MODIFY age DEFAULT NULL;
CHECK

The CHECK constraint is used to limit the value range that can be placed in a column.

While Creating the Table
create table stu(
name varchar2(10) not null,
studid varchar2(5),
degreelevel varchar2(15),
primary key(studid),
check(degreelevel in('Bachelors','Masters','Doctorate'))
);
Adding to exisiting table
ALTER TABLE Persons ADD CONSTRAINT CHK_PersonAge CHECK (Age>=18 AND City='Sandnes');
Droping the Contraint
ALTER TABLE Persons DROP CONSTRAINT CHK_PersonAge;

Truncate

TRUNCATE TABLE table_name
Drop vs Truncate
  1. Truncate is normally ultra-fast and its ideal for deleting data from a temporary table.
  2. Truncate preserves the structure of the table for future use, unlike drop table where the table is deleted with its full structure.
  3. Table or Database deletion using DROP statement cannot be rolled back, so it must be used wisely.

Select

SELECT column1,column2 FROM table_name
Select Distinct
SELECT DISTINCT column1, column2, ... FROM table_name;
Count the distinct values
SELECT COUNT( DISTINCT column_name) FROM table_name;
WHERE clause
SELECT column1, column2, ... FROM table_name WHERE condition;
Possible operators in where
  • =
  • <>
  • >=
  • <=
  • BETWEEN
  • LIKE
  • IN
  • AND
  • OR
  • NOT

Note: AND, OR, NOT can be combined with brakets.

The LIKE Command
  • % - The percent sign represents zero, one, or multiple characters
  • _ - The underscore represents a single character

Note: LIKE can be combined with NOT, AND, OR. Example: SELECT * FROM table_name WHERE column_name NOT LIKE 'a%';

ORDER BY
SELECT column1, column2, ... FROM table_name ORDER BY column1, column2, ... ASC|DESC;
For several Columns
SELECT * FROM Customers ORDER BY Country ASC, CustomerName DESC;

Insert

Values only:
INSERT INTO table_name VALUES (value1, value2, value3,...);
Columns and Values:
INSERT INTO table_name (column1, column2, column3,..) VALUES ( value1, value2, value3,..);
Insert from another Table
Copy all columns
INSERT INTO first_table SELECT * FROM second_table;
Copy specific columns
INSERT INTO first_table(column1, column2, ...) SELECT column1, column2, ... FROM second_table;
Copy with condition
INSERT INTO table1 SELECT * FROM table2 WHERE condition;

Update

UPDATE table_name SET column1 = value1, column2 = value2,... WHERE condition;

Note: If you miss to put the where command, all tuples will be updated!

Delete

DELETE FROM table_name WHERE some_condition;

Note: If you miss to put the where command, all tuples will be deleted!

Delete All Tuples
DELETE FROM table_name;
